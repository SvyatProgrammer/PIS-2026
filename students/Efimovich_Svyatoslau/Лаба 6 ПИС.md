## Цель работы

Создать комплексную стратегию тестирования для системы **“Да, я в деле”**, включающую юнит-тестирование домена и бизнес-логики, интеграционное тестирование базы данных и сквозное (E2E) тестирование API.

**Вариант — Сервис заявок “Да, я в деле”**

* Ядро домена: агрегат `ParticipationRequest`, объекты-значения `UserId`, `EventId`.
* Технологический стек: ASP.NET Core, xUnit, Moq, Testcontainers (PostgreSQL).

---

## Ход выполнения работы

### 1. Юнит-тесты (Domain Layer)

**Файл:** `ParticipationRequestTests.cs`
**Покрытие:** бизнес-правила агрегата.

**Реализованные сценарии:**

* Проверка инварианта: `UserId` и `EventId` не могут быть пустыми.
* Проверка начального состояния: заявка создаётся в статусе `Pending`.
* Проверка правил: нельзя повторно создать заявку на одно и то же событие.
* Проверка регистрации доменного события `RequestCreatedEvent`.

**Пример:**

```csharp
[Fact]
public void Should_Throw_When_UserId_Is_Empty()
{
    Assert.Throws<ArgumentException>(() =>
        new UserId(Guid.Empty));
}
```

---

### 2. Юнит-тесты (Application Layer)

**Файл:** `CreateParticipationRequestHandlerTests.cs`
**Инструменты:** Moq

Протестирован `CreateParticipationRequestHandler` с использованием моков:

* `IParticipationRequestRepository`
* `IEventPublisher`

**Проверяется:**

* Вызов `SaveAsync()`
* Публикация доменных событий
* Возврат корректного `Id`

**Пример:**

```csharp
[Fact]
public async Task Should_Save_Request_And_Publish_Events()
{
    var repoMock = new Mock<IParticipationRequestRepository>();
    var publisherMock = new Mock<IEventPublisher>();

    var handler = new CreateParticipationRequestHandler(
        repoMock.Object,
        publisherMock.Object);

    var command = new CreateParticipationRequestCommand(Guid.NewGuid(), Guid.NewGuid());

    var result = await handler.Handle(command);

    repoMock.Verify(r => r.SaveAsync(It.IsAny<ParticipationRequest>()), Times.Once);
    publisherMock.Verify(p => p.Publish(It.IsAny<object>()), Times.AtLeastOnce);
}
```

### 3. Интеграционные тесты (Infrastructure Layer)

**Файл:** `ParticipationRequestRepositoryTests.cs`
**Технология:** Testcontainers (PostgreSQL)

В тестах используется реальная база данных, поднятая в контейнере.

**Проверяется:**

* Сохранение заявки в БД
* Корректное чтение
* Восстановление состояния агрегата

**Сценарий:**

1. Создать объект `ParticipationRequest`
2. Сохранить через репозиторий
3. Получить обратно
4. Сравнить значения

---

### 4. E2E-тесты (Сквозной сценарий)

**Файл:** `ParticipationRequestApiTests.cs`
**Инструменты:** `WebApplicationFactory`, `HttpClient`

**Сценарий:**

1. `POST /api/requests`

   * Отправка JSON с `UserId` и `EventId`
   * Получение `201 Created`

2. `GET /api/requests/{id}`

   * Получение данных заявки

**Пример:**

```csharp
[Fact]
public async Task FullFlow_Should_Work()
{
    var client = _factory.CreateClient();

    var command = new
    {
        userId = Guid.NewGuid(),
        eventId = Guid.NewGuid()
    };

    var postResponse = await client.PostAsJsonAsync("/api/requests", command);
    postResponse.EnsureSuccessStatusCode();

    var location = postResponse.Headers.Location;

    var getResponse = await client.GetAsync(location);
    getResponse.EnsureSuccessStatusCode();
}
```
