## Ход выполнения работы

### 1. Repository (PostgreSQL + EF Core)

**Технологии:** Entity Framework Core, PostgreSQL.

Реализован адаптер `ParticipationRequestRepository`, который является реализацией интерфейса `IParticipationRequestRepository`.

Для хранения данных используется сущность `ParticipationRequestEntity`.

**Реализованные методы:**

* `SaveAsync(ParticipationRequest)` — сохранение агрегата в БД
* `GetByIdAsync(Guid)` — получение агрегата по ID

**Пример:**

```csharp
public class ParticipationRequestRepository : IParticipationRequestRepository
{
    private readonly AppDbContext _context;

    public ParticipationRequestRepository(AppDbContext context)
    {
        _context = context;
    }

    public async Task SaveAsync(ParticipationRequest request)
    {
        var entity = new ParticipationRequestEntity
        {
            Id = request.Id,
            UserId = request.UserId.Value,
            EventId = request.EventId.Value,
            Status = request.Status.ToString()
        };

        _context.Requests.Add(entity);
        await _context.SaveChangesAsync();
    }

    public async Task<ParticipationRequest?> GetByIdAsync(Guid id)
    {
        var entity = await _context.Requests.FindAsync(id);

        if (entity == null) return null;

        return ParticipationRequest.Restore(
            entity.Id,
            new UserId(entity.UserId),
            new EventId(entity.EventId),
            Enum.Parse<RequestStatus>(entity.Status)
        );
    }
}
```

**Схема таблицы:**
Таблица `participation_requests`:

* `id` (UUID)
* `user_id` (UUID)
* `event_id` (UUID)
* `status` (VARCHAR)

### 2. REST API Controller

Реализован входящий адаптер — Web API контроллер.

**Эндпоинты:**

| Метод | Path                 | Описание        |
| ----- | -------------------- | --------------- |
| POST  | `/api/requests`      | Создать заявку  |
| GET   | `/api/requests/{id}` | Получить заявку |

**Пример:**

```csharp
[ApiController]
[Route("api/requests")]
public class ParticipationRequestController : ControllerBase
{
    private readonly CreateParticipationRequestHandler _createHandler;
    private readonly GetParticipationRequestHandler _getHandler;

    public ParticipationRequestController(
        CreateParticipationRequestHandler createHandler,
        GetParticipationRequestHandler getHandler)
    {
        _createHandler = createHandler;
        _getHandler = getHandler;
    }

    [HttpPost]
    public async Task<IActionResult> Create(CreateParticipationRequestCommand command)
    {
        var id = await _createHandler.Handle(command);
        return CreatedAtAction(nameof(GetById), new { id }, null);
    }

    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(Guid id)
    {
        var result = await _getHandler.Handle(new GetParticipationRequestQuery(id));
        return Ok(result);
    }
}
```

## Контрольные вопросы

**1. Почему Repository находится в Infrastructure?**
Потому что он зависит от конкретной технологии (EF Core, PostgreSQL). Домен должен быть независим от этих деталей.

---

**2. В чём преимущество EF Core?**
Позволяет работать с БД как с объектами, автоматически управляет миграциями и уменьшает количество ручного SQL.
